# Implementation Sequence

1. Owner approves services, areas, minimums, contact facts, claims/proof, offers, capacity and lifecycle definitions.
2. Legal/privacy review approves consent, tags, DNI/call recording stance, retention and vendors.
3. Select owner-controlled hosting/form, CRM/system of record, analytics/tag and call architecture—no secrets in repo.
4. Create development/staging environment and test-only measurement destinations.
5. Implement secure server form/upload pipeline, IDs, validation, spam/rate limits, error/thank-you states.
6. Configure CRM fields, stages, reasons, permissions and routing; train reviewers.
7. Implement consent controls, attribution persistence, data layer and primary/secondary events.
8. Implement call tracking/DNI with fallback, classification and missed-call process.
9. Implement offline stage exports/imports and correction/deduplication controls.
10. Run functional, security, privacy, mobile, accessibility, performance, link/metadata and reconciliation QA.
11. Obtain written owner approval and record launch checklist. Only then may separate authorized work publish pages/connect services; Phase 4 ads stay paused until every gate passes.
