# Mobile QA Checklist

Test at 320, 360, 390, 768, and 1024 CSS px and on at least one real iOS and Android device.

- [ ] No horizontal overflow at 320 px or 200% zoom.
- [ ] H1/CTA visible without overlap; no text embedded in images.
- [ ] Tap targets are at least 44×44 CSS px with spacing.
- [ ] Call and estimate actions remain distinguishable and do not obstruct content.
- [ ] Navigation wraps/works by touch and keyboard; skip link works.
- [ ] Forms use appropriate input types/autocomplete, readable labels, and inline errors.
- [ ] Software keyboard does not hide active control/error; focus scroll is sensible.
- [ ] Upload control and filenames fit; orientation changes preserve state.
- [ ] FAQ summaries and form tabs work with keyboard/screen reader.
- [ ] Placeholder aspect ratios prevent layout shift.
- [ ] No hover-only information; reduced-motion preference respected.
- [ ] Test slow connection, failed asset, thank-you/error routes, and back navigation.
- [ ] Owner validates real call number, hours, coverage and mobile task completion before launch.
