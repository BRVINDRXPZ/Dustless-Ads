## Program standard
This is a service protocol, not a membership claim or priority guarantee.
1. **Onboard:** fit, geography, project types, buying roles, payment/contract norms, communication preference and proof need.
2. **Bid:** use bid intake; acknowledge receipt only after review; identify gaps; perform plan/site review when triggered; issue inclusions, exclusions, alternates, assumptions, validity and dependencies subject to owner-approved proposal policy.
3. **Mobilize:** signed authorization/contract/PO, deposit/payment gate if applicable, selections, site readiness, material, access, schedule and other trades confirmed.
4. **Execute:** named contacts; agreed update rhythm; daily condition/progress notes as appropriate; stop/escalate surprises; no extra work without documented authorization except immediate safety control.
5. **Close:** punch route, photos/records with permission, care/product documents when applicable, warranty language only if approved, invoice/waiver requirements and lessons.
6. **Grow:** post-project review, quarterly review for active accounts, suitable referrals and next-project planning.

Priority handling means prompt internal review relative to declared capacity; it never guarantees a slot. Referral compensation, warranty, care, dispute and escalation language require owner/legal approval. Escalation: field contact → project lead → owner; document issue, impact, options, decision owner and deadline.
