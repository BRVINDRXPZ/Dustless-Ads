# GBP Posting System

## Workflow
1. Select an approved category from the calendar; confirm the relevant GBP post feature still exists.
2. Create a factual brief: audience, one useful point, service, area only if permissioned, CTA and page.
3. Attach asset ID and written permission; remove faces/plates/addresses/documents; confirm same-project before/after and color accuracy.
4. Claims reviewer marks facts **verified**, **qualified**, or **remove**. Customer/review/neighborhood stories require explicit permission.
5. Add controlled `utm_source=google&utm_medium=organic&utm_campaign=gbp_<theme>_2026q3&utm_content=<asset-token>` only after Phase 5 implementation approval; no PII.
6. Owner previews copy/image/link, validates current requirements and explicitly approves. Publisher records URL/date; this phase stops before that step.
7. QA at publication and archive outcome/lead-quality observations monthly without claiming attribution from correlation.

## Recurring categories and guardrails
Project transformation; Can this floor be saved; floor-care education; historic-floor insight; contractor update; commercial capability; service explanation; seasonal scheduling; process transparency; permissioned review/customer story; permissioned neighborhood story; team/equipment; FAQ; before-and-after reveal. Historic/commercial/customer/location claims stay blocked until proof/permission.

## Caption frame
**Observation:** what the viewer can actually see. **Education:** one expert-reviewed, qualified point. **Process:** what was performed, only from job record. **Next step:** low-pressure CTA. Never fake urgency, offers, results, dust-free performance, customer speech, location or availability.

## Cadence and ownership
Calendar proposes one draft/week for 13 weeks; cadence is not a claim about platform preference. Content lead drafts; operations verifies; permissions owner clears assets; owner approves; authorized profile manager may later publish. Pause if capacity, routing, permissions, policy warnings or factual uncertainty arise.
