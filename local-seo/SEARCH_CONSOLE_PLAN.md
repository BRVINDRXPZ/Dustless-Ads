# Search Console Plan

## Access and setup (future, approval-gated)
Verify owner-controlled Domain property via the approved DNS administrator; use least privilege and 2FA; record users/recovery without secrets. Also map canonical HTTPS variants where useful. Confirm site ownership before sitemap action. Do not connect under Phase 6.

## Initial audit
1. Validate sitemap URL/content, canonicals, status codes and intended index set before submission.
2. Review Page Indexing reasons and sample URL Inspection rendered/canonical data.
3. Export page/query/device/country/date performance with freshness/anonymization limitations documented.
4. Segment city/service and “near me” query patterns only from observed data; do not infer precise searcher location.
5. Identify competing pages for same queries; inspect intent before consolidation.
6. Find CTR opportunities only where impressions/position/intent are sufficient; never promise CTR outcomes.
7. Review Core Web Vitals field data and page groups; use lab diagnostics separately.
8. Review Enhancements applicable to actual markup, Manual Actions and Security Issues immediately.
9. Analyze internal/external Links as directional samples, not a complete backlink index.

## Monthly report
Reporting window and prior-year/comparable period; clicks/impressions/CTR/average position with caveats; branded/nonbranded rule; service/local query groups; landing pages; newly indexed/excluded; cannibalization cases; CWV; enhancements; manual/security status; link observations; GBP/CRM qualified outcomes where safely joinable; actions/owner/deadline. Annotate site/GBP changes and major anomalies.

## Governance
Never export PII; restrict raw exports; retain by approved policy; distinguish sampled/anonymized data; owner approves fixes and publication. Search Console is diagnostic—not a complete rank or revenue system.
