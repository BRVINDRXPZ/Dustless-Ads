# Map Rank Tracking Plan

## Baseline
Before any approved change, use a clean documented browser state while acknowledging localization/personalization cannot be eliminated. Track a fixed query/location/device matrix at a consistent weekday/time. Locations must represent owner-approved markets, not customer addresses. Capture screenshots privately and record exact observation—not an averaged invented “rank.”

## Register schema
`query`, `location_name`, `approved non-sensitive coordinate/grid token`, `device`, `browser/state`, `date_time_utc`, `organic results observed`, `local pack results observed`, `NOLA Dustless position (integer/not visible/not measured)`, `competitors observed`, `ads present`, `personalized-state warning`, `screenshot path`, `notes`, `auditor`.

## Query set (validate with Search Console/research later)
Hardwood floor refinishing, hardwood floor sanding, hardwood floor installation, hardwood floor repair, wood floor contractor, plus proof-approved historic/heart-pine/commercial variants and city modifiers. Do not treat these as measured volume.

## Procedure
1. Record device, account/sign-in state, location method, consent and browser; avoid VPN/spoofed locations that violate tool/platform terms.
2. Capture full results, local pack and organic, ads presence and timestamp. Competitors are dated observations, not stable facts.
3. Repeat manually monthly for core points; annotate GBP/site/category/review/content changes and seasonality.
4. Compare visibility coverage and qualified lead outcomes, not rank alone. Never cherry-pick points.
5. Later evaluate a reputable grid tool only after terms, cost, data handling, accuracy and owner approval; retain a manual control point.

No baseline was taken in Phase 6 and no current position is claimed.
