# GBP Photography System

## Standards
| Asset | Standard |
|---|---|
| Hero/cover | real permissioned flagship project; uncluttered; safe crop at multiple ratios; no text-heavy promotion; validate current specs |
| Logo | approved master, correct clear space/color; test circular/square crops |
| Crew | releases, PPE and safe behavior; no customer/private data |
| Equipment/sanding | clean, truthful setup; identify only verified equipment/process; never imply “dust free” |
| Installation/repair/historic/commercial | show actual documented scope; historic/commercial labels require proof; conceal plans, badges and addresses |
| Before/after | same project, room, lens, angle and broadly comparable light; label sequence; no compositing or removed defects |
| Vertical/horizontal video | stable establishing/process/detail/reveal; capture both orientations when safe; no copyrighted audio; captions later |
| Color/editing | neutral white balance; exposure/crop/straighten only; no material color swaps, object fabrication, misleading cleanup or AI alteration |
| Privacy/geolocation | strip GPS by default; never reveal customer address, alarm, documents, plates, minors or neighborhood without permission |
| Cadence | capture every eligible project; propose a steady upload queue only after quality/permission review and capacity approval |

**Naming:** `YYYYMMDD_project-token_service_phase_view_orientation_v01.ext`; token is non-identifying. Keep private job ID linkage only in the approved restricted asset system—not filenames/public metadata.

**Asset register fields:** asset ID, restricted project ID, capture date, service, general area (permissioned), phase, orientation, photographer, rights owner, people/property/customer permissions, privacy review, edit log, technical accuracy, approved channels, expiry/revocation, public URL, status. Do not store customer addresses here.

## Every-project shot list
| Phase | Required shots | QA |
|---|---|---|
| 1 Arrival | tools/vehicle only where safe; room establishing | no address/plate; property permission |
| 2 Existing condition | wide, medium, detail, consistent reference angle | honest light/color |
| 3 Damage details | scale/context and multiple angles | no diagnosis until expert confirms |
| 4 Prep | protection, containment, staged workspace | safe practice; private items removed |
| 5 Sanding | wide process and equipment interface | PPE; no performance exaggeration |
| 6 Edging | edge process/detail | safety and technical review |
| 7 Repair | sequence, material, fit before finish | species/match claim verified |
| 8 Stain/finish prep | samples only with permission; clean substrate | label sample vs final; color caveat |
| 9 Finish application | process and wet-state context | product/coat facts from job record |
| 10 Final reveal | reference-angle pair, wide vertical/horizontal | cure/access rules followed |
| 11 Details | grain, transitions, stairs, repair | representative, not defect-hiding |
| 12 Crew/equipment | portrait/action and setup | releases/PPE/branding |
| 13 Exterior/context | only customer-approved, non-identifying context | explicit location/exterior permission |

## Transfer and approval
Back up originals; preserve immutable master and edit derivative; malware-scan; restrict job linkage; obtain written channel-specific permission; conduct privacy, claim and quality review; revoke/withdraw according to the permission record. Nothing enters a public upload queue without owner approval.
