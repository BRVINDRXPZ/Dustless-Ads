# CRM Implementation Plan

> **Planning specification — inactive.** No live accounts, credentials, communications, or real customer data.

## Purpose

Vendor-neutral migration path.

## Immediate low-complexity option

Use a controlled Google Sheets-style interim register (category recommendation, not a feature/pricing claim) with protected ranges, IDs, validation, access control and backups while requirements are proven. Owner must approve actual platform and privacy.

## Best long-term category

Evaluate a CRM/job-management stack capable of residential job flow plus flexible account/opportunity pipelines and owned export/API access. No named vendor is recommended until current features, pricing, security and integrations are verified.

## Migration

Inventory → decide source of truth → configure sandbox → map/version → clean/dedupe → dry run → reconcile → user acceptance → backup/rollback → approved cutover → monitor. Customer-facing automations remain off.

## Decision criteria

Workflow fit across all segments; estimate/job needs; outcome attribution; finance reconciliation; permissions/privacy; exports/API; usability/mobile; total cost; implementation burden; vendor continuity. Score only with dated evidence and owner weighting.
