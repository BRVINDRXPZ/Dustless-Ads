# Content Approval Workflow
> **Internal draft only.** No publication, outreach, live connection, spend, or asset use is authorized. Verify claims and record permissions before use.

## RACI and gates
| Gate | Accountable check | Reject/hold if |
|---|---|---|
| Source | Capture lead | Missing source ID, backup, or metadata |
| Rights/privacy | Permission owner | Release unclear; private detail; copyrighted media/music |
| Facts | Owner/technical reviewer | Species, history, product, process, result, service, schedule, credential, cost unverified |
| Brand | Content lead | Hype, jargon, filler, competitor/customer humiliation, off-brand visual |
| Operations | Owner/ops | CTA cannot be handled; availability/service area unconfirmed |
| Final | Owner-designated approver | Any prior gate incomplete |
| Publish | Authorized publisher | No explicit per-item publish authorization or preflight failure |

## Approval record
Content ID; version/hash; source asset IDs; exact caption/cover/export; platforms; fact sources; permissions and expiry; reviewer/date at every gate; CTA destination; timing; community owner; final authorization. Silence is never approval.

## Corrections/withdrawals
Pause queued derivatives, preserve evidence, notify owner, correct or remove through the authorized account operator, log reason/date/actions, identify all reused versions, update proof register. Complaints and legal/privacy issues are never handled casually in comments.
