# Command Center Implementation Roadmap

> **Planning specification — not connected or activated.** No live accounts, credentials, customer data, or invented business performance. Any prototype values are synthetic demonstration data.

## Purpose

Sequenced to earn data trust before sophistication.

## Roadmap

| Phase | Scope | Exit gate |
|---|---|---|
| A — Minimum viable control | Lead register, source, qualification, estimate stages, win/loss, revenue, gross profit, weekly report | Required fields and reconciled sample-free real internal process approved |
| B — Marketing integration | Forms, call tracking, Ads, analytics, GBP, Search Console, social, reviews | Access/consent/QA approval; read-only imports tie out |
| C — Sales and B2B | Contractor accounts, commercial opportunities, activities, follow-up, bids, account management | Owners use stages consistently; aging/actions reliable |
| D — Automation | Internal alerts, reminders, review workflows, reporting, data QA, reviewed offline conversion imports | Per-automation approval, tests, rollback, audit log |
| E — Optimization | Attribution, conversion-value bidding, forecasting, capacity, CLV, concentration, marketing mix | Clean longitudinal data and owner-approved models; no false precision |

## Phase A checklist

Choose interim register; assign owners; import no unreviewed PII; train stages; run weekly reconciliation; baseline missingness; obtain finance definition of revenue and GP.

## Stop gates

No next phase if duplicate IDs, stage misuse, unreconciled outcomes, hidden Unknown, access/privacy, capacity, or owner approval fails.
