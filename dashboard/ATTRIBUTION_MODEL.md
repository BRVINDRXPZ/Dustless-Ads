# Attribution Model

> **Planning specification — not connected or activated.** No live accounts, credentials, customer data, or invented business performance. Any prototype values are synthetic demonstration data.

## Purpose

Practical, auditable credit without false precision.

## Models

**First touch:** earliest verified discovery. **Last touch:** last verified acquisition touch before lead. **Lead source:** normalized source at lead creation. **Opportunity source:** source that created the sales opportunity, preserved on conversion. **Revenue source:** primary opportunity source joined to recognized revenue. **GP attribution:** same source joined only after actual GP approval. **Assisted:** other verified touches, non-additive. **Contractor/referral:** named referral relationship plus originating channel. **Offline:** human-captured “how heard,” call records, or event IDs. **Unknown:** explicit Unknown, never redistributed.

## Practical default

For a small local service business, use immutable first touch plus a governed **primary lead/opportunity source** as the default reporting model. Attribute reconciled revenue and GP to that primary source; report verified assists separately without fractional credit. Preserve customer-stated referral and digital evidence, and show conflicts.

## Multiple sources

Do not manufacture fractional precision. Select primary via documented evidence and human review; retain all touches. Shared projects may be reported once at project level with an attribution note.

## Coverage

Every report shows known-source coverage and Unknown. Compare models only as sensitivity analysis; never sum assisted revenue with primary-source revenue.
